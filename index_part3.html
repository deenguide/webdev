Section 7: GitHub Terms

Code সংরক্ষণ ও team-এর সাথে কাজ করার platform

▼
01

Repository

রিপোজিটরি / কোড ঘর ▼
📖
বাংলা অর্থ
একটি project-এর সমস্ত files, folders ও history সংরক্ষণের জায়গা — সংক্ষেপে "Repo"
💡
সহজ ব্যাখ্যা
GitHub-এ একটি Repo = একটি project-এর সম্পূর্ণ folder। সব code, image, file সেখানে থাকে। Public = সবাই দেখতে পারে। Private = শুধু তুমি।
🐙 উদাহরণ
# নতুন repo তৈরি করা
git init

# GitHub-এর repo clone করা
git clone https://github.com/aminulislam79/deen-app.git
🌍 বাস্তব উদাহরণ

তোমার DEEN App-এর সব code GitHub-এ একটি Repository-তে আছে। URL: github.com/aminulislam79/deen-app — এটাই তার Repo।

02

Commit

কমিট / পরিবর্তন সংরক্ষণ ▼
📖
বাংলা অর্থ
Code-এর পরিবর্তন একটি message সহ permanently save করা
💡
সহজ ব্যাখ্যা
বইয়ের নতুন chapter লিখে save করার মতো। প্রতিটি Commit-এ কী পরিবর্তন হয়েছে সেটার একটা message থাকে। History দেখা যায়।
🐙 উদাহরণ
git add .                          ← সব পরিবর্তন stage করো
git commit -m "Login page তৈরি করা হলো"
↑ message দিয়ে permanently save
03

Push

পুশ / আপলোড ▼
📖
বাংলা অর্থ
Local computer-এর commit গুলো GitHub server-এ পাঠানো
💡
সহজ ব্যাখ্যা
Commit = নিজের computer-এ save। Push = সেই save GitHub-এ upload। Push না করলে internet-এ code যাবে না।
🐙 উদাহরণ
git push origin main
↑ "main" branch-এ push করো

# GitHub Pages deploy হয় push করলেই!
04

Pull

পুল / ডাউনলোড ▼
📖
বাংলা অর্থ
GitHub server থেকে সর্বশেষ পরিবর্তন local computer-এ আনা
💡
সহজ ব্যাখ্যা
Push-এর উল্টো। Team member কিছু push করলে তুমি pull করে নিজের computer-এ নতুন code আনো। সবাই sync-এ থাকো।
🐙 উদাহরণ
git pull origin main
↑ GitHub থেকে সর্বশেষ code নামাও

# Push = উপরে পাঠাও ↑
# Pull = নিচে আনো   ↓
05

Branch

ব্রাঞ্চ / শাখা ▼
📖
বাংলা অর্থ
Main code থেকে আলাদা কপি তৈরি করে নতুন feature develop করা
💡
সহজ ব্যাখ্যা
গাছের ডালের মতো। Main (trunk) = মূল code। Branch = আলাদা ডালে কাজ করো, main code নষ্ট হওয়ার ভয় নেই।
🐙 উদাহরণ
git branch new-feature    ← নতুন branch তৈরি
git checkout new-feature   ← সেই branch-এ যাও

# Default branch-এর নাম: main বা master
06

Merge

মার্জ / মেলানো ▼
📖
বাংলা অর্থ
একটি branch-এর পরিবর্তন অন্য branch-এ (সাধারণত main-এ) যোগ করা
💡
সহজ ব্যাখ্যা
নতুন feature তৈরি শেষ হলে সেটা main code-এ মিশিয়ে দেওয়া। নদীর ছোট শাখা মূল নদীতে মেলার মতো।
🐙 উদাহরণ
git checkout main            ← main-এ যাও
git merge new-feature        ← feature যোগ করো
↑ এখন main-এ নতুন feature আছে!
07

Fork

ফোর্ক / কপি করা ▼
📖
বাংলা অর্থ
অন্য কারো GitHub repository নিজের account-এ copy করা
💡
সহজ ব্যাখ্যা
কারো recipe দেখে নিজের খাতায় copy করে নিজের মতো বদলানো = Fork। Original-এ হাত লাগে না, তুমি নিজের copy-তে কাজ করো।
🌍 বাস্তব উদাহরণ

GitHub-এ কোনো open-source project পছন্দ হলে Fork করো → নিজের account-এ চলে আসবে → নিজের মতো বদলাও। Original project-এর মালিক কিছু জানবে না।

08

Clone

ক্লোন / নামিয়ে আনা ▼
📖
বাংলা অর্থ
GitHub-এর কোনো repository সম্পূর্ণভাবে নিজের computer-এ download করা
💡
পার্থক্য
Fork = GitHub-এ GitHub-এ copy। Clone = GitHub থেকে তোমার PC-তে download। কাজ করতে হলে Clone করতেই হবে।
🐙 উদাহরণ
git clone https://github.com/aminulislam79/amal-checklist.git

↑ এই command চালালে পুরো project
  তোমার PC-তে নেমে আসবে!
🌐

Section 8: Hosting Terms

Website internet-এ publish করার সব প্রয়োজনীয় ধারণা

▼
01

Domain

ডোমেইন / ঠিকানা ▼
📖
বাংলা অর্থ
Internet-এ website-এর নামের ঠিকানা — যেমন google.com, facebook.com
💡
সহজ ব্যাখ্যা
বাড়ির ঠিকানা না থাকলে কেউ আসতে পারবে না। Domain হলো website-এর ঠিকানা। .com, .org, .bd — এগুলো domain extension।
🌍 বাস্তব উদাহরণ

তোমার DEEN App-এর address: aminulislam79.github.io/deen-app — এটাই তার Domain (GitHub Pages-এর sub-domain)। নিজের domain কিনলে হতো: deenapp.com

02

Hosting

হোস্টিং / জায়গা ভাড়া ▼
📖
বাংলা অর্থ
Website-এর files internet-এ সবসময় চালু থাকা server-এ রাখার service
💡
সহজ ব্যাখ্যা
দোকান দিতে জায়গা ভাড়া নিতে হয় — Hosting হলো internet-এ সেই জায়গা। GitHub Pages, Vercel, Netlify = free hosting।
🌍 বাস্তব উদাহরণ

তোমার সব projects GitHub Pages-এ free host হচ্ছে। Paid hosting-এ Namecheap, GoDaddy, Hostinger পাওয়া যায়। বাংলাদেশে: ExonHost, Host.com.bd।

03

DNS

ডিএনএস / নামের অনুবাদক ▼
📖
বাংলা অর্থ
Domain Name System — নামকে (google.com) IP address (142.250.x.x)-এ রূপান্তর করে
💡
সহজ ব্যাখ্যা
ফোনের contact list-এ নাম লেখা থাকে, number না। DNS-ও তেমনি — "google.com" দিলে সে জানে কোন server-এ যেতে হবে।
🌍 বাস্তব উদাহরণ

তুমি browser-এ "facebook.com" লেখো → DNS server জিজ্ঞেস করো "এর IP কত?" → DNS বলে "157.240.x.x" → Browser সেই IP-তে যায় → Facebook খোলে।

04

SSL / HTTPS

এসএসএল / নিরাপদ সংযোগ ▼
📖
বাংলা অর্থ
Secure Sockets Layer — browser ও server-এর মধ্যে data encrypt করে পাঠানো
💡
সহজ ব্যাখ্যা
URL-এর শুরুতে 🔒 বা "https" দেখলে SSL আছে। Password বা card number পাঠালে কেউ মাঝপথে চুরি করতে পারবে না।
🌍 বাস্তব উদাহরণ

http:// = অরক্ষিত, ডাকপিওন চিঠি পড়তে পারে। https:// = চিঠি sealed envelope-এ, কেউ খুলতে পারবে না। GitHub Pages-এ SSL free দেওয়া থাকে।

05

CDN

সিডিএন / দ্রুত বিতরণ ▼
📖
বাংলা অর্থ
Content Delivery Network — বিশ্বজুড়ে server-এ file রেখে user-এর কাছাকাছি থেকে দ্রুত দেওয়া
💡
সহজ ব্যাখ্যা
ঢাকায় বসে USA-র server থেকে image load করলে slow হয়। CDN ঢাকার কাছের server থেকে দেয় — তাই fast।
🌍 বাস্তব উদাহরণ

এই page-এ Google Fonts load হচ্ছে CDN থেকে। Cloudflare, jsDelivr, cdnjs — এগুলো popular CDN। Bootstrap, jQuery সব CDN থেকে ব্যবহার করা যায়।

06

Deployment

ডেপ্লয়মেন্ট / প্রকাশ করা ▼
📖
বাংলা অর্থ
Local বা GitHub-এর code internet-এ live করে সবার জন্য accessible করা
💡
সহজ ব্যাখ্যা
রান্না করা = code লেখা। খাবার table-এ পরিবেশন করা = Deployment। যতক্ষণ deploy না হয় কেউ দেখতে পারবে না।
🌍 বাস্তব উদাহরণ

তোমার DEEN App GitHub-এ push করলেই GitHub Pages automatically deploy করে। Vercel ও Netlify-তেও auto-deploy হয়। এটাই CI/CD-এর ধারণা।

07

Static Website

স্ট্যাটিক ওয়েবসাইট ▼
📖
বাংলা অর্থ
শুধু HTML, CSS, JavaScript দিয়ে তৈরি — server-side কোনো processing নেই
💡
সহজ ব্যাখ্যা
Static = স্থির। সবার কাছে একই content যায়। Database নেই, login নেই। Fast, cheap, secure। GitHub Pages = Static hosting।
🌍 বাস্তব উদাহরণ

তোমার এই Dictionary file, DEEN App, Amal Checklist — সবই Static Website। শুধু HTML file, GitHub Pages-এ বিনামূল্যে host হচ্ছে।

08

Dynamic Website

ডায়নামিক ওয়েবসাইট ▼
📖
বাংলা অর্থ
Server-এ processing হয়, database থেকে data আসে — প্রতিটি user-এর জন্য আলাদা content
💡
সহজ ব্যাখ্যা
Facebook-এ তুমি login করলে তোমার feed, আমি login করলে আমার feed। একই website কিন্তু আলাদা content — এটাই Dynamic।
🌍 বাস্তব উদাহরণ

Facebook, YouTube, Daraz, Pathao — সব Dynamic Website। Login, user data, real-time update সব আছে। Backend ও Database লাগে।

Section 9: AI Coding Terms

AI দিয়ে coding করার যুগের গুরুত্বপূর্ণ ধারণাগুলো

▼
01

ChatGPT

চ্যাটজিপিটি ▼
📖
বাংলা অর্থ
OpenAI-এর তৈরি AI chatbot — text লিখে যেকোনো প্রশ্নের উত্তর পাওয়া যায়
💡
সহজ ব্যাখ্যা
GPT = Generative Pre-trained Transformer। Code লেখা, debugging, explanation, translation — সব করতে পারে। chat.openai.com-এ free ব্যবহার করা যায়।
🌍 তুমি কীভাবে ব্যবহার করো

তুমি ChatGPT দিয়ে code লেখো, idea নাও, content তৈরি করো। এটা তোমার AI সহকর্মী।

02

Claude

ক্লড ▼
📖
বাংলা অর্থ
Anthropic-এর তৈরি AI assistant — দীর্ঘ ও জটিল কাজে বিশেষভাবে দক্ষ
💡
সহজ ব্যাখ্যা
Claude বড় file পড়তে পারে, দীর্ঘ code লিখতে পারে, detailed ব্যাখ্যা দেয়। claude.ai-তে free ব্যবহার করা যায়।
🌍 তুমি কীভাবে ব্যবহার করো

তুমি ৩টি Gmail account দিয়ে ৩টি Claude account ব্যবহার করো — এই পুরো Dictionary project Claude দিয়েই তৈরি হচ্ছে!

03

Gemini

জেমিনি ▼
📖
বাংলা অর্থ
Google-এর AI model — text, image, code সব বোঝে ও তৈরি করতে পারে
💡
সহজ ব্যাখ্যা
Gemini বিশেষভাবে image generation-এ ভালো। Google-এর সব products-এ integrate হচ্ছে। gemini.google.com-এ free।
🌍 তুমি কীভাবে ব্যবহার করো

তুমি Gemini দিয়ে cinematic short film ও reels-এর জন্য image generate করো, তারপর image-to-video tool-এ দাও।

04

Prompt

প্রম্পট / নির্দেশ ▼
📖
বাংলা অর্থ
AI-কে দেওয়া নির্দেশ বা প্রশ্ন — যা লিখে দিলে AI সেই অনুযায়ী কাজ করে
💡
সহজ ব্যাখ্যা
ভালো Prompt = ভালো Result। AI কে যত clearly বলবে কী চাও, তত ভালো কাজ করবে। এটাকে "Prompt Engineering" বলে।
🤖 ভালো vs খারাপ Prompt
❌ খারাপ: "একটা website বানাও"

✅ ভালো: "একটি বাংলা Recipe website
বানাও — mobile friendly, dark mode সহ,
Noto Serif Bengali font, ৫টি recipe card"
05

Token

টোকেন ▼
📖
বাংলা অর্থ
AI যেভাবে text পড়ে ও লেখে তার সবচেয়ে ছোট একক — মোটামুটি ১ token ≈ ৩-৪ অক্ষর বা ৭৫% word
💡
সহজ ব্যাখ্যা
AI পুরো word এক সাথে না পড়ে ভেঙে ভেঙে পড়ে। Token = সেই ভাঙা টুকরো। API-তে token count অনুযায়ী charge হয়।
🌍 বাস্তব উদাহরণ

"Programming" = ৩ টোকেন। "আমি" = ২ টোকেন। Claude-এর free plan-এ প্রতি conversation-এ limited token পাওয়া যায়। বড় project ভাগ করে করতে হয়।

06

Context Window

কনটেক্সট উইন্ডো / স্মৃতির সীমা ▼
📖
বাংলা অর্থ
AI একটি conversation-এ কতটুকু text একসাথে মনে রাখতে পারে তার সীমা
💡
সহজ ব্যাখ্যা
মানুষ একটি বইয়ের ২০ পাতা মনে রাখতে পারে। Context Window হলো AI-এর সেই সীমা। এর বাইরে গেলে আগের কথা ভুলে যায়।
🌍 বাস্তব উদাহরণ

Claude-এর সাথে conversation অনেক বড় হলে শুরুর কথা ভুলে যায় — এটাই Context Window-এর সীমা। তাই বড় project ভাগ করে নতুন conversation-এ করা ভালো।

07

AI Agent

এআই এজেন্ট / স্বায়ত্তশাসিত AI ▼
📖
বাংলা অর্থ
AI যে নিজে নিজে plan করে, tools ব্যবহার করে এবং কাজ সম্পন্ন করে
💡
সহজ ব্যাখ্যা
Normal AI = প্রশ্নের উত্তর দেয়। AI Agent = নিজে search করে, code চালায়, file বানায়, সিদ্ধান্ত নেয়। একটি digital কর্মচারী।
🌍 বাস্তব উদাহরণ

তুমি বললে "আমার জন্য একটি portfolio website বানাও, GitHub-এ push করো এবং deploy করো" — AI Agent সব নিজে করবে। Claude Code = AI Coding Agent।

08

MCP

এমসিপি / AI টুল সংযোগ ▼

বাংলা অর্থ
Model Context Protocol — AI-কে বাইরের tools ও services-এর সাথে সংযুক্ত করার standard পদ্ধতি
💡
সহজ ব্যাখ্যা
AI = brain। MCP = হাত ও পা। MCP দিয়ে AI Google Search করতে পারে, file পড়তে পারে, calendar দেখতে পারে। Anthropic তৈরি করেছে।
🌍 বাস্তব উদাহরণ

Claude-এ web search, calendar, file create — এগুলো সব MCP-র মাধ্যমে কাজ করছে। এই page যখন Claude বানাচ্ছে তখন file create MCP ব্যবহার করছে।

Section 10: Beginner Roadmap

একদম শূন্য থেকে Professional Developer হওয়ার Step-by-Step পথ

▼

🎯 লক্ষ্য: এই roadmap follow করলে তুমি ৬-১২ মাসে একজন AI-assisted Web Developer হতে পারবে। প্রতিটি step-এ real project বানাও — শুধু পড়া নয়, হাতে করো।

1

HTML শেখো

প্রথম ধাপ
HTML হলো সব কিছুর ভিত্তি। Website-এর skeleton বানানো শেখো। Tag, Element, Attribute, Form, Table — সব মুখস্থ না করে বুঝে বুঝে শেখো।

🎯 Project: নিজের Personal Profile Page বানাও — ছবি, নাম, ঠিকানা, hobbies দিয়ে।
⏱️ সময়: ১-২ সপ্তাহ 🆓 Free Resource: W3Schools 🤖 AI Help: Claude/ChatGPT
2

CSS শেখো

দ্বিতীয় ধাপ
HTML দিয়ে কাঠামো, CSS দিয়ে সৌন্দর্য। Color, Margin, Padding, Flexbox, Grid, Responsive Design শেখো। Mobile-first design করো।

🎯 Project: Profile Page-কে সুন্দর করো। Dark mode যোগ করো। Mobile-এ ভালো দেখায় করো।
⏱️ সময়: ২-৩ সপ্তাহ 🆓 Free Resource: CSS-Tricks 🎨 Tool: Flexbox Froggy
3

JavaScript শেখো

তৃতীয় ধাপ
JavaScript দিয়ে website interactive হয়। Variable, Function, Event, DOM, Fetch API, LocalStorage শেখো। সব মুখস্থ না করলেও চলে — concept বোঝো।

🎯 Project: To-Do App বানাও — add, delete, LocalStorage-এ save।
⏱️ সময়: ৩-৪ সপ্তাহ 🆓 Free Resource: JavaScript.info 🤖 AI Help: Claude দিয়ে debug
4

GitHub শেখো

চতুর্থ ধাপ
Code সংরক্ষণ ও share করার সবচেয়ে গুরুত্বপূর্ণ tool। Repository, Commit, Push, Pull শেখো। GitHub account তৈরি করো।

🎯 Project: আগের সব project GitHub-এ upload করো। GitHub Pages-এ live করো।
⏱️ সময়: ১ সপ্তাহ 🆓 Hosting: GitHub Pages (Free) 📱 App: GitHub Mobile
5

Hosting বোঝো

পঞ্চম ধাপ
Domain, Hosting, DNS, SSL, Deployment বোঝো। Free hosting দিয়ে শুরু করো — GitHub Pages, Vercel, Netlify। নিজের project-এর link সবাইকে দিতে পারবে।

🎯 Project: নিজের একটি project Vercel-এ deploy করো।
⏱️ সময়: ১ সপ্তাহ 🆓 Free: Vercel / Netlify 💰 Domain: .com ≈ ১২০০ টাকা/বছর
6

AI-Assisted Coding

ষষ্ঠ ধাপ
ChatGPT, Claude, Gemini দিয়ে দ্রুত code লেখা শেখো। ভালো Prompt লিখতে শেখো। AI-এর দেওয়া code বুঝতে পারো — শুধু copy-paste নয়।

🎯 Project: AI দিয়ে একটি complete Web App বানাও — Task Manager বা Recipe App।
⏱️ সময়: Ongoing 🤖 Tools: Claude + ChatGPT 💡 Key: Prompt Engineering
7

Real Projects বানাও

চূড়ান্ত ধাপ
শেখা শেষ হয় না — Real project-ই সবচেয়ে ভালো শিক্ষক। নিজের জীবনের সমস্যা সমাধানের জন্য app বানাও। Portfolio তৈরি করো। অন্যদের কাজে লাগে এমন কিছু বানাও।
